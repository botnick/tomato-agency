# คู่มือเขียนบทความ Tomato Agency (Content + SEO + Humanize)

บทความทุกชิ้นต้อง **อ่านเหมือนคนเขียน ไม่ใช่ AI** — เป็นมิตร เฉพาะเจาะจง และจริงใจ ห้ามก๊อปจากที่อื่น ห้ามใช้ AI gen รูป

## เพิ่มบทความใหม่ (ง่าย ๆ ไฟล์เดียว)

1. สร้างไฟล์ `src/blog/<slug>.md` (slug เป็นภาษาอังกฤษ คั่นด้วย `-`)
2. ใส่ front matter:

```yaml
---
title: "หัวข้อบทความ"
description: "สรุป 1–2 ประโยค ใส่คีย์เวิร์ดหลัก (ใช้เป็น meta description + การ์ด)"
date: 2026-06-24
category: "เริ่มต้นเป็นวีเจ"      # ชื่อหมวด (โชว์)
categorySlug: "start"             # slug หมวด (ใช้ทำหน้า /blog/category/<slug>/)
cover: /assets/img/blog/<slug>.png
coverAlt: "คำอธิบายรูปสั้น ๆ"
tagsList: ["คีย์เวิร์ด1", "คีย์เวิร์ด2"]
faq:                              # ไม่บังคับ — ใส่แล้วได้ FAQ schema อัตโนมัติ
  - q: "คำถาม"
    a: "คำตอบ"
---
```

3. เขียนเนื้อหาด้วย Markdown (`## หัวข้อ`, `-` ลิสต์, ลิงก์ภายในไปหน้า keyword)
4. สร้าง cover: เปิด `scripts/og-gallery.html` เพิ่มรายการใหม่ใน `COVERS` แล้ว screenshot เป็น PNG 1200×630 ลง `assets/img/blog/<slug>.png` (ดู README หัวข้อ "OG covers")
5. `npm run build` แล้ว preview — sitemap/RSS/หน้า category อัปเดตเอง

## โทนเสียง (ตามกฎแบรนด์)

- Live VJ คือบริการหลักเสมอ — ไม่รีโพสเป็นค่ายไอดอล/ศิลปิน (IDOL+ = ชื่อโปรแกรมพัฒนาวีเจ)
- เรียกทีม **"พี่ ๆ ทีมงาน"** เรียกผู้สมัคร **"น้อง ๆ"** ใช้คำลงท้ายนุ่ม ๆ ได้ (นะ จ้า)
- **ห้ามกุตัวเลข/รายได้/สถิติ** ไม่การันตีรายได้
- เฉพาะเจาะจง เล่าจากประสบการณ์จริง ดีกว่าพูดกว้าง ๆ

## ห้ามใช้ (กลิ่น AI / แปลแข็ง)

ยินดีต้อนรับสู่ · ในยุคดิจิทัล · อย่างไรก็ตาม · นวัตกรรม · โซลูชัน · ปฏิวัติวงการ ·
อย่างไร้ที่ติ · ในบทความนี้เราจะ · โดยสรุปแล้ว · ตอบโจทย์ทุกความต้องการ · ครบวงจรที่สุด

- [ ] มี cover 1200×630 + `coverAlt`
